## 기능 1. 운동별 진행 차트

### 개요
특정 운동의 중량/볼륨 변화를 시간순 그래프로 시각화하여 성장 추이를 확인하는 기능.

### 진입 경로
- 캘린더 화면 → 상단 AppBar에 "통계" 아이콘 버튼 추가 → 통계 화면 이동

### 화면 구성: ExerciseStatsScreen

```
┌─────────────────────────────┐
│  ←  운동 통계                │
├─────────────────────────────┤
│                             │
│  [운동 선택 드롭다운 ▼]      │
│   벤치프레스                 │
│                             │
│  기간: [4주][8주][12주][전체] │
│                             │
│  ┌───────────────────────┐  │
│  │                    ·  │  │
│  │              ·   ·    │  │
│  │         ·  ·          │  │
│  │    ·  ·               │  │
│  │  ·                    │  │
│  └───────────────────────┘  │
│   1/6  1/13  1/20  1/27     │
│                             │
│  [최고 중량] [총 볼륨] 탭    │
│                             │
│  ── 기록 요약 ──────────── │
│  최고 중량: 85kg (2/15)     │
│  최근 중량: 80kg            │
│  총 수행 횟수: 24회          │
│  처음 기록: 2026-01-05      │
│                             │
└─────────────────────────────┘
```

### 차트 데이터 종류 (탭 전환)
| 탭 | X축 | Y축 | 데이터 포인트 |
|---|---|---|---|
| 최고 중량 | 날짜 | kg | 해당 날짜 해당 운동의 max(weight) |
| 총 볼륨 | 날짜 | kg | 해당 날짜 해당 운동의 sum(weight × reps) |

### 기간 필터
- 4주 / 8주 / 12주 / 전체
- 기본값: 8주

### 기록 요약 영역
차트 하단에 텍스트로 표시:
- 역대 최고 중량 + 달성 날짜
- 가장 최근 기록 중량
- 해당 운동 총 수행 일수
- 최초 기록 날짜

### 데이터 조회

기존 테이블로 조회 가능. 새 테이블 불필요.

```
-- 특정 운동의 날짜별 최고 중량
SELECT dw.date, MAX(ws.weight) as maxWeight
FROM daily_workouts dw
JOIN workout_records wr ON wr.dailyWorkoutId = dw.id
JOIN workout_sets ws ON ws.workoutRecordId = wr.id
WHERE wr.exerciseId = :exerciseId
  AND dw.date >= :startDate
GROUP BY dw.date
ORDER BY dw.date ASC

-- 특정 운동의 날짜별 총 볼륨
SELECT dw.date, SUM(ws.weight * ws.reps) as totalVolume
FROM daily_workouts dw
JOIN workout_records wr ON wr.dailyWorkoutId = dw.id
JOIN workout_sets ws ON ws.workoutRecordId = wr.id
WHERE wr.exerciseId = :exerciseId
  AND dw.date >= :startDate
GROUP BY dw.date
ORDER BY dw.date ASC
```

## 기능 2. 개인 기록(PR) 자동 감지

### 개요
운동 저장 시 해당 운동의 역대 최고 중량을 갱신했는지 자동 체크하고, 갱신 시 축하 UI를 표시.

### 동작 흐름

```
[세트 편집 화면에서 "완료" 클릭]
        │
        ▼
[WorkoutSetViewModel.saveAndFinish()]
        │
        ▼
[저장된 세트 중 max(weight) 추출]
        │
        ▼
[해당 exerciseId의 역대 max(weight) 조회]  ← 오늘 데이터 제외
        │
        ├── 역대 기록 없음 (첫 기록) → PR 표시하지 않음
        ├── 역대 기록 ≥ 현재 → 변동 없음
        └── 역대 기록 < 현재 → PR 달성!
                │
                ▼
        [PR 결과를 UI State에 반영]
                │
                ▼
        [DailyWorkoutScreen에서 PR 다이얼로그/스낵바 표시]
```

### PR 판정 기준
- **대상**: 세트 중 최고 중량 (weight 필드)
- **비교**: 해당 운동의 오늘 이전 역대 최고 중량
- **첫 기록**: PR로 표시하지 않음 (비교 대상 없음)

### PR 표시 UI

#### 방안: 다이얼로그
세트 편집 완료 후 DailyWorkoutScreen 복귀 시 다이얼로그 표시:
```
┌─────────────────────────┐
│                         │
│     PR 달성!            │
│                         │
│   벤치프레스             │
│   85kg → 90kg           │
│   (+5kg)                │
│                         │
│        [ 확인 ]         │
└─────────────────────────┘
```

여러 운동에서 동시에 PR 발생 가능성은 낮지만 (세트 편집은 운동별로 진입), 혹시 복사 기능 등으로 발생 시 마지막 PR만 표시하거나 리스트로 표시.

```kotlin
// saveAndFinish() 수정
fun saveAndFinish() {
    viewModelScope.launch {
        // 1. 기존 세트 저장 로직 (변경 없음)
        saveSets()

        // 2. PR 체크 (추가)
        val currentMax = _uiState.value.sets.maxOfOrNull { it.weight } ?: 0f
        val exerciseId = _uiState.value.workoutRecord?.exercise?.id ?: return@launch
        val todayDate = _uiState.value.workoutRecord?.let { /* dailyWorkout date */ }

        val previousMax = statsDao.getExerciseMaxWeightBefore(exerciseId, todayDate)

        if (previousMax != null && currentMax > previousMax) {
            _uiState.update { it.copy(
                prResult = PrResult(
                    exerciseName = it.workoutRecord!!.exercise.name,
                    exerciseCategory = it.workoutRecord!!.exercise.category,
                    previousMax = previousMax,
                    newMax = currentMax
                )
            ) }
        }

        // 3. 화면 종료
        _uiState.update { it.copy(isDone = true) }
    }
}
```

## 기능 3. 주간/월간 요약 통계

### 개요
캘린더 화면에서 현재 보고 있는 월의 운동 요약 통계를 표시. 운동 일수, 총 볼륨, 부위별 비중 등.

### 진입 경로
캘린더 화면 하단에 요약 카드로 상시 표시 (접기/펼치기 가능).

### 화면 구성: CalendarScreen 하단 추가

### 표시 항목

#### 월간 기본 통계
| 항목 | 계산 |
|---|---|
| 운동 일수 | 해당 월 daily_workouts 레코드 수 |
| 총 볼륨 | 해당 월 모든 세트의 sum(weight × reps) |
| 총 세트 수 | 해당 월 모든 workout_sets 수 |

#### 부위별 볼륨 비중
| 항목 | 계산 |
|---|---|
| 카테고리별 볼륨 | 카테고리별 sum(weight × reps) |
| 비중(%) | 카테고리 볼륨 / 총 볼륨 × 100 |
- 수평 바 차트 + 퍼센트 텍스트
- 볼륨 높은 순 정렬
- 기록 없는 카테고리는 표시하지 않음

#### 주간 볼륨 비교
| 항목 | 계산 |
|---|---|
| 주별 볼륨 | 해당 주(월~일) 모든 세트의 sum(weight × reps) |
- 해당 월에 포함되는 주차별로 표시 (보통 4~5주)
- 수평 바 차트

### 접기/펼치기
- 기본 상태: 펼침
- 접으면 "2월 요약" 헤더만 보임
- 상태는 ViewModel에서 관리 (앱 재시작 시 초기화 OK)

### 필요한 DAO 쿼리 추가

```kotlin
// StatsDao

data class MonthlyStats(
    val workoutDays: Int,
    val totalVolume: Float,
    val totalSets: Int
)

data class CategoryVolumeStat(
    val category: String,
    val volume: Float
)

data class WeeklyVolumeStat(
    val weekStart: Long,
    val volume: Float
)

// 월간 기본 통계
@Query("""
    SELECT
        COUNT(DISTINCT dw.id) as workoutDays,
        COALESCE(SUM(ws.weight * ws.reps), 0) as totalVolume,
        COUNT(ws.id) as totalSets
    FROM daily_workouts dw
    LEFT JOIN workout_records wr ON wr.dailyWorkoutId = dw.id
    LEFT JOIN workout_sets ws ON ws.workoutRecordId = wr.id
    WHERE dw.date >= :startDate AND dw.date <= :endDate
""")
suspend fun getMonthlyStats(startDate: Long, endDate: Long): MonthlyStats

// 부위별 볼륨
@Query("""
    SELECT e.category, SUM(ws.weight * ws.reps) as volume
    FROM daily_workouts dw
    JOIN workout_records wr ON wr.dailyWorkoutId = dw.id
    JOIN exercises e ON wr.exerciseId = e.id
    JOIN workout_sets ws ON ws.workoutRecordId = wr.id
    WHERE dw.date >= :startDate AND dw.date <= :endDate
    GROUP BY e.category
    ORDER BY volume DESC
""")
suspend fun getCategoryVolumeInRange(startDate: Long, endDate: Long): List<CategoryVolumeStat>
```

## 기능 4. 운동 선택 시 최근 기록 표시

### 개요
운동 추가를 위해 운동 선택 BottomSheet에서 각 운동 항목 옆에 해당 운동의 가장 최근 기록(날짜 + 세트 요약)을 간결하게 표시하여 참고할 수 있게 하는 기능.

### 표시 형식
```
벤치프레스
가슴 · 3/1 32.5:12/32.5:12/32.5:12
```
- `월/일` : 마지막으로 해당 운동을 수행한 날짜
- `무게:횟수` : 각 세트의 무게와 반복수를 콜론으로 연결
- 세트 간 구분자: `/`

### 데이터 조회

기존 테이블로 조회 가능. 새 테이블/마이그레이션 불필요.

```sql
-- 각 운동별 가장 최근 workout_record ID와 날짜
SELECT wr.exerciseId, wr.id as recordId, dw.date
FROM workout_records wr
JOIN daily_workouts dw ON wr.dailyWorkoutId = dw.id
WHERE wr.id = (
    SELECT wr2.id
    FROM workout_records wr2
    JOIN daily_workouts dw2 ON wr2.dailyWorkoutId = dw2.id
    WHERE wr2.exerciseId = wr.exerciseId
    ORDER BY dw2.date DESC, wr2.id DESC
    LIMIT 1
)
```

이후 각 recordId에 대한 세트 정보를 batch로 조회.
