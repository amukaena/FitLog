# FitLog - VSCode 개발 환경 설정

## 1. 설치 현황 ✅

| 항목 | 경로 | 상태 |
|------|------|------|
| JDK 17 | `D:\00_JDK\jdk-17` | ✅ 설치됨 |
| Android SDK | `D:\Android` | ✅ 설치됨 |
| Command Line Tools | `D:\Android\cmdline-tools\latest` | ✅ 설치됨 |
| Platform Tools (ADB) | `D:\Android\platform-tools` | ✅ 설치됨 |
| Build Tools 36.0.0 | `D:\Android\build-tools\36.0.0` | ✅ 설치됨 |
| Android Platform 36 | `D:\Android\platforms\android-36` | ✅ 설치됨 |

## 2. 환경 변수 설정 (필수)

**PowerShell (관리자 권한)에서 실행:**

```powershell
# JAVA_HOME 설정
[Environment]::SetEnvironmentVariable("JAVA_HOME", "D:\00_JDK\jdk-17", "User")

# ANDROID_HOME 설정
[Environment]::SetEnvironmentVariable("ANDROID_HOME", "D:\Android", "User")

# Path에 추가
$currentPath = [Environment]::GetEnvironmentVariable("Path", "User")
$newPaths = "D:\00_JDK\jdk-17\bin;D:\Android\cmdline-tools\latest\bin;D:\Android\platform-tools"
[Environment]::SetEnvironmentVariable("Path", "$currentPath;$newPaths", "User")
```

**설정 후 VSCode/터미널 재시작 필요**

## 2. 설치 확인

PowerShell에서 다음 명령어 실행:

```powershell
# JDK 확인
java -version

# Android SDK 확인
sdkmanager --version

# ADB 확인 (기기 연결용)
adb version
```

## 3. 기기 연결 (USB 디버깅)

1. **개발자 옵션 활성화**
   - 설정 > 휴대전화 정보 > 빌드 번호 7번 탭

2. **USB 디버깅 켜기**
   - 설정 > 개발자 옵션 > USB 디버깅 ON

3. **연결 확인**
   ```powershell
   adb devices
   # 기기가 목록에 나타나면 성공
   ```

## 4. 빌드 & 실행 명령어

프로젝트 루트에서:

```powershell
# 빌드
./gradlew assembleDebug

# APK 설치 & 실행
./gradlew installDebug

# 또는 직접 설치
adb install app/build/outputs/apk/debug/app-debug.apk
```
